---
title: "Quick Start"
description: "Install OpenSpec and run your first spec-driven change in under five minutes."
---

By the end of this guide you will have OpenSpec installed, initialized in a project, and a working `/opsx:propose` workflow running in your AI tool.

<Steps>
  <Step title="Install OpenSpec">
    **Requires Node.js 20.19.0 or higher.**

    ```bash
    npm install -g @fission-ai/openspec@latest
    ```

    Verify the install:

    ```bash
    openspec --version
    ```
  </Step>

  <Step title="Initialize your project">
    Navigate to your project directory and run:

    ```bash
    cd your-project
    openspec init
    ```

    The interactive setup asks which AI tools you use, then installs the appropriate slash commands and skills. If you want to skip the prompt, pass `--tools` directly:

    ```bash
    openspec init --tools claude,cursor
    ```

    After init, you will have an `openspec/` directory at the root of your project.

    <Tip>
      Not sure which tools are supported? OpenSpec works with 20+ AI assistants including Claude Code, Cursor, GitHub Copilot, Windsurf, Gemini CLI, and more. See the [supported tools reference](/integrations/supported-tools) for the full list.
    </Tip>
  </Step>

  <Step title="Propose a change">
    Open your AI tool and run:

    ```text
    /opsx:propose add-my-feature
    ```

    Your AI will create a change folder with all the planning artifacts:

    ```text
    You: /opsx:propose add-dark-mode
    AI:  Created openspec/changes/add-dark-mode/
         ✓ proposal.md — why we're doing this, what's changing
         ✓ specs/       — requirements and scenarios
         ✓ design.md    — technical approach
         ✓ tasks.md     — implementation checklist
         Ready for implementation!
    ```

    Review the generated artifacts. You can edit any of them before moving on — update the proposal, refine the specs, or adjust the task list. The AI will work from whatever is in those files.
  </Step>

  <Step title="Apply the change">
    Once you are happy with the plan, tell your AI to implement it:

    ```text
    /opsx:apply
    ```

    Your AI works through the task checklist in `tasks.md` one item at a time:

    ```text
    You: /opsx:apply
    AI:  Implementing tasks...
         ✓ 1.1 Add theme context provider
         ✓ 1.2 Create toggle component
         ✓ 2.1 Add CSS variables
         ✓ 2.2 Wire up localStorage
         All tasks complete!
    ```

    If you discover something needs to change mid-implementation, just update the artifact and continue. There are no phase gates.
  </Step>

  <Step title="Archive the change">
    When implementation is complete, archive the change:

    ```text
    /opsx:archive
    ```

    OpenSpec merges the delta specs into your main `openspec/specs/` directory and moves the change folder to `openspec/changes/archive/`:

    ```text
    You: /opsx:archive
    AI:  Archived to openspec/changes/archive/2025-01-23-add-dark-mode/
         Specs updated. Ready for the next feature.
    ```
  </Step>
</Steps>

## What just happened?

You ran the core OpenSpec workflow:

| Step | Command | What it does |
|------|---------|---------------|
| Propose | `/opsx:propose` | Creates a change folder with proposal, specs, design, and task checklist |
| Apply | `/opsx:apply` | Implements the task checklist from `tasks.md` |
| Archive | `/opsx:archive` | Merges delta specs into main specs and moves change to history |

The `openspec/specs/` directory is now your source of truth — a growing record of how your system behaves, updated automatically with every archived change.

<Note>
  OpenSpec works best with high-reasoning models. Opus 4.5 and GPT 5.2 are recommended for both planning and implementation.
</Note>
