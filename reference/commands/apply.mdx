title: "/opsx:apply"
description: "Implement tasks from the change, working through the task checklist and writing code."
---

`/opsx:apply` reads the `tasks.md` file in a change and implements each task one by one — writing code, creating files, and running tests as needed. Progress is tracked via checkboxes in `tasks.md`.

## Syntax

```text
/opsx:apply [change-name]
```

## Arguments

<ParamField path="change-name" type="string">
  The name of the change to implement. If not provided, the AI infers the current change from context.
</ParamField>

## What it does

- Reads `tasks.md` and identifies incomplete tasks (unchecked `[ ]` items)
- Works through tasks one by one in order
- Writes code, creates files, and runs tests as needed
- Marks each task complete with a checkbox `[x]` as it finishes
- Can be interrupted and resumed — completion state is tracked in `tasks.md`

## Example

```text
You: /opsx:apply

AI:  Implementing add-dark-mode...

     Reading tasks.md:
     - [ ] 1.1 Create ThemeContext
     - [ ] 1.2 Add CSS custom properties
     - [ ] 1.3 Implement localStorage persistence

     Working on 1.1: Create ThemeContext...
     [Creates src/context/ThemeContext.tsx]

     ✓ 1.1 Complete

     Working on 1.2: Add CSS custom properties...
     [Updates src/styles/globals.css]

     ✓ 1.2 Complete
     ...
```

## Tips

- If implementation is interrupted, run `/opsx:apply` again to resume where you left off
- When working on parallel changes, specify the change name explicitly: `/opsx:apply add-dark-mode`
- Completion state is tracked directly in `tasks.md` checkboxes, so progress survives session restarts
- Run `/opsx:verify` after apply to check implementation quality before archiving
