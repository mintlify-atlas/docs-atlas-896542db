---
title: "/opsx:archive"
description: "Archive a completed change, optionally merging delta specs into main specs."
---

`/opsx:archive` finalizes a completed change by checking artifact status, offering to sync delta specs, and moving the change folder to the archive directory. Archived changes are preserved as a permanent audit trail.

## Syntax

```text
/opsx:archive [change-name]
```

## Arguments

<ParamField path="change-name" type="string">
  The name of the change to archive. If not provided, the AI infers the current change from context.
</ParamField>

## What it does

- Checks artifact completion status
- Checks task completion in `tasks.md` (warns if tasks are incomplete, but does not block)
- Offers to sync delta specs into `openspec/specs/` if not already synced
- Moves the change folder to `openspec/changes/archive/YYYY-MM-DD-<name>/`
- Preserves all artifacts for audit trail

## Example

```text
You: /opsx:archive

AI:  Archiving add-dark-mode...

     Artifact status:
     ✓ proposal.md exists
     ✓ specs/ exists
     ✓ design.md exists
     ✓ tasks.md exists (8/8 tasks complete)

     Delta specs: Not yet synced
     → Sync now? (recommended)

You: Yes

AI:  ✓ Synced specs to openspec/specs/ui/spec.md
     ✓ Moved to openspec/changes/archive/2025-01-24-add-dark-mode/

     Change archived successfully.
```

## Tips

- Archive will warn about incomplete tasks but won't block — you can archive a partially complete change
- Delta specs can be synced during archive or beforehand with `/opsx:sync`
- Archived changes are preserved permanently and can be referenced for history
- Run `/opsx:verify` before archiving to catch implementation issues early
- To archive multiple changes at once, use `/opsx:bulk-archive` (expanded workflow)
