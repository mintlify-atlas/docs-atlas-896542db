OpenSpec works with many AI coding assistants. Running `openspec init` configures your selected tools using your active profile and delivery mode.

## How it works

For each selected tool, OpenSpec can install up to two types of integration files:

1. **Skills** (if delivery includes skills): `SKILL.md` files placed in the tool's skills directory. Skills are passive — the AI reads them to understand available workflows.
2. **Commands** (if delivery includes commands): `opsx-*` command files placed in the tool's commands directory. Commands are invocable as slash commands or prompts.

You can configure delivery mode to install skills only, commands only, or both.

## Delivery modes

| Mode | What gets installed | When to use |
|------|---------------------|-------------|
| **Both** (default) | Skills + command files | Most tools |
| **Skills only** | Only `SKILL.md` files | Tools without command support |
| **Commands only** | Only command files | When you prefer explicit invocation |

## Workflow profiles

The **core profile** (default) installs four workflows:

- `propose`
- `explore`
- `apply`
- `archive`

To install expanded workflows (`new`, `continue`, `ff`, `verify`, `sync`, `bulk-archive`, `onboard`), configure your profile first:

```bash
openspec config profile
# then run:
openspec update
```

## Tool-specific notes

<Note>
  **Codex** — Command files are installed in the global Codex home directory (`$CODEX_HOME/prompts/` if set, otherwise `~/.codex/prompts/`), not in your project directory.
</Note>

<Note>
  **GitHub Copilot** — Prompt files (`.github/prompts/*.prompt.md`) are recognized as custom slash commands in IDE extensions (VS Code, JetBrains, Visual Studio). Copilot CLI does not currently consume these files directly.
</Note>

<Note>
  **Trae** — No command adapter is generated. Use skill-based `/openspec-*` invocations instead.
</Note>

## Non-interactive setup

For CI/CD or scripted environments, pass `--tools` to skip the interactive prompt:

```bash
# Configure specific tools
openspec init --tools claude,cursor

# Configure all 24 supported tools
openspec init --tools all

# Skip tool configuration entirely
openspec init --tools none

# Override the workflow profile for this run
openspec init --profile core
```

**All available tool IDs:**

```text
amazon-q, antigravity, auggie, claude, cline, codex, codebuddy, continue,
costrict, crush, cursor, factory, gemini, github-copilot, iflow, kilocode,
kiro, opencode, pi, qoder, qwen, roocode, trae, windsurf
```

## Workflow-dependent installation

The number of skill and command files installed depends on your active profile. OpenSpec only installs files for the workflows you've enabled.

| Profile | Workflows installed |
|---------|---------------------|
| Core (default) | `propose`, `explore`, `apply`, `archive` |
| Custom | Any subset of all workflow IDs |

All available workflow IDs: `propose`, `explore`, `new`, `continue`, `apply`, `ff`, `sync`, `archive`, `bulk-archive`, `verify`, `onboard`
