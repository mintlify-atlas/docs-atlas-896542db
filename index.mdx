## What is OpenSpec?

OpenSpec adds a lightweight spec layer between you and your AI coding assistant. Instead of starting with vague prompts and hoping for good results, you agree on *what* to build before any code is written.

```text
You: /opsx:propose add-dark-mode
AI:  Created openspec/changes/add-dark-mode/
     ✓ proposal.md — why we're doing this, what's changing
     ✓ specs/       — requirements and scenarios
     ✓ design.md    — technical approach
     ✓ tasks.md     — implementation checklist
     Ready for implementation!

You: /opsx:apply
AI:  Implementing tasks...
     ✓ 1.1 Add theme context provider
     ✓ 1.2 Create toggle component
     ✓ 2.1 Add CSS variables
     ✓ 2.2 Wire up localStorage
     All tasks complete!

You: /opsx:archive
AI:  Archived to openspec/changes/archive/2025-01-23-add-dark-mode/
     Specs updated. Ready for the next feature.
```

## Why OpenSpec?

AI coding assistants are powerful but unpredictable when requirements live only in chat history. OpenSpec brings predictability without adding ceremony.

## How It Works

<Steps>
  <Step title="Install OpenSpec">
    Install the CLI globally with npm:
    ```bash
    npm install -g @fission-ai/openspec@latest
    ```
  </Step>
  <Step title="Initialize your project">
    Navigate to your project and run:
    ```bash
    openspec init
    ```
    This sets up the `openspec/` folder structure and configures your AI tools.
  </Step>
  <Step title="Propose a change">
    In your AI assistant's chat interface, run:
    ```text
    /opsx:propose add-my-feature
    ```
    Your AI will create a proposal, specs, design, and tasks — all before writing any code.
  </Step>
  <Step title="Implement and archive">
    Run `/opsx:apply` to implement the tasks, then `/opsx:archive` to finalize the change and merge specs.
  </Step>
</Steps>
